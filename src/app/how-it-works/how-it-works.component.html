How It Works

Overview

My website is a Static Hosted Website on S3 built using Angular in a GitHub Repo. To deploy this app in AWS the following services will be used: * Code Pipeline * Code Build * S3 Bucket

AWS - S3

Alot of directions here are pulled from https://docs.aws.amazon.com/AmazonS3/latest/dev/website-hosting-custom-domain-walkthrough.html

Create S3 Bucket

To create your buckets for website hosting
  1. Sign in to the AWS Management Console and open the Amazon S3 console at https://console.aws.amazon.com/s3/.
  2. Create your root domain:
    1. Choose Create bucket. ex: MyAppBucket
    2. Enter the Bucket name (for example, example.com).
    3. Choose the Region where you want to create the bucket.
    4. To accept the default settings and create the bucket, choose Create.

Configure S3 Bucket

To host a static website on Amazon S3, an Amazon S3 bucket has to be configured for website hosting. To configure your bucket for website hosting

To configure your bucket for website hosting

  1. On the Amazon S3 console, in the S3 buckets list, choose the created for this project. ex: MyAppBucket
  2. Choose Properties.
  3. Choose Static website hosting.
  4. Choose Use this bucket to host a website.
  5. In the Index Document box, enter index.html Note If you don't enter the name of your index page, the Save button won't become active, and you won't be able to configure your bucket for static website hosting.

AWS - CodePipeline

To create a CodePipeline Pipeline

  1. Sign in to the AWS Management Console and open the CodePipeline console at https://console.aws.amazon.com/codesuite/codepipeline/pipelines
  2. Create CodePipeline
    1. Click Create Pipeline and fill out the Pipeline Name; Leave the Service Role default selection
    2. For the Source Stage, select GitHub
    3. After Connecting to GitHub, pick the GitHub Repo created, and select Master as the branch. Leave the default. Change detection options
    4. For the Build Stage, select the AWS CodeBuild as the Build Provider
    5. Select a region. Note: The region selected should the same as the region the bucket exists
    6. For the project name, click Create Project

AWS - CodeBuild
